Key Concepts on HTML and CSS

Lesson 1: Basics of HTML and the Web

Major pieces of the web: Browser -> Internet -> HTTP Request to Server -> The the same in reverse HTML - Hypertext markup language, main web doc type:
  • Heart of the web
  • Made up of text content - what you see
  • Markup - what it looks like
  • Can include references to - images, videos etc
  • Other docs
  • Links

HTML Markup - made up of tags such as b for bold, em for emphasis or italics. Image tags (img) do not include a closing tag, these are called void tags. Good coding calls for alternate text to be provided for images as well.

HTML elements are either inline, or block. Block elements form an invisible box around the content inside them. Examples of inline elements are b, img or span. P, div and others are examples of block elements.

HTML attributes are part of HTML elements, for example using the href attribute to provide the URL in an anchor tag.

Lesson 2: Adding Structure to HTML Docs

  • HTML and CSS are both languages
  • HTML controls the "structure" of a web page
  • CSS controls the "style" of a page (how it looks). Cascading Style Sheets - style sheet language used for describing the look and formatting of a document written in a markup language.
  • When programmers talk about the "DOM" (Document Object Model) they are talking about the tree-like structure of a page. The tree structure allows elements to have other elements inside of them.
  • Structure is important when coding your HTML documents as it allows for easy review of your work. An example of this would be to add a logical structure, such as a setting up one a set of styles that can be easily reused for your purpose. Time is saved for everyone - the person coding and the end user who will get a page to load more quickly.
  • Properly nested code will have identantions which will go deeper and deeper the more nexted they are within their box.
  • Inspecting elements and developer tools in your browser can be a huge time saver for troubleshooting and design direction.

Lesson 3: Adding CSS to HTML

  • CSS code will allow you to process repeated formatting quickly and easily. It can be applied to the entire sheet, an identified class or even elements.
  • HTML controls the "structure" of a web page
  • CSS controls the "style" of a page (how it looks). Cascading Style Sheets - style sheet language used for describing the look and formatting of a document written in a markup language. Although you can apple style directly in your HTML doc, that is only recommended for very short lines of code. A standalone CSS file is the best route.
  • The item you will be "styling" is called a "selector". The declationis what you are modifying (font size, color) to which value (for example red font, size 12). The following website provides reference materials on this CSS Guide
  • CSS works by applying a major rule when it comes to "cascading". It is a rule of inheritance - it basically states that properties assigned to a specific element will not only apply to the specified element, but also to it's descendants. This website explains the rule in detail
  • Flexbox allows for easily manipulation of boxes for alignment. This site provides the ins and outs of how it works
  • Always keep in mind the 4 box componenets (margin, border, padding and content). Sizing by percentage is almost always a better bet than pixels to help account for varied resolutions.
  • When coding, look for natural boxes, repeated styles/semantic elements, write the HTML, apply styles(from biggest to smallest) and then fix things. Repeat 3-5 as neccesary.
